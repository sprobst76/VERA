## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | p-1 | Icon-to-label gap, badge inner padding |
| sm | 8px | p-2 | Compact cell padding, tag gaps |
| xs-plus | 12px | py-3 | Table cell vertical padding |
| md | 16px | p-4 | Default element spacing, card inner padding |
| lg | 24px | p-6 | Section gaps, filter bar padding |
| xl | 32px | p-8 | Layout column gaps |
| 2xl | 48px | p-12 | Major section breaks |
| 3xl | 64px | p-16 | Page-level top spacing |

Table row cells use `px-4 py-3` — 16px horizontal (md) and 12px vertical (xs-plus). Matches existing compliance and payroll table patterns exactly.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 |
| Label | 12px (text-xs) | 400 (regular) | 1.4 |
| Heading | 24px (text-2xl) | 700 (bold) | 1.2 |
| Table header | 12px (text-xs) | 400 (regular) | 1.4 |

Source: Observed in `compliance/page.tsx` — `text-2xl font-bold` for page heading, `text-xs text-muted-foreground` for table headers, `text-sm` for body rows. The 12px vs 14px size contrast provides sufficient visual separation between table headers/labels and body text without requiring a distinct weight.

Note: Only 3 distinct sizes are declared (12px, 14px, 24px). The before/after diff values inside expanded rows use 12px monospace (`font-mono`) to distinguish data from prose.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `rgb(var(--background))` — Latte: #eff1f5 / Mocha: #1e1e2e | Page background |
| Secondary (30%) | `rgb(var(--card))` — Latte: #e6e9ef / Mocha: #181825 | Cards, filter bar, table container, sidebar |
| Accent (10%) | `rgb(var(--ctp-blue))` — Latte: #1e66f5 / Mocha: #89b4fa | See reserved-for list below |

Accent reserved for:
- The "Audit-Log" active nav item highlight in the sidebar
- The count/total badge when results are loaded (e.g. "42 Einträge")
- Focus ring on filter inputs (`ring` token = `var(--ring)` = ctp-blue)

Semantic color map for action-type badges (read from `action` field on each log row):

| Action | Color token | Background |
|--------|-------------|------------|
| create | `--ctp-green` | `rgb(var(--ctp-green) / 0.12)` |
| update | `--ctp-blue` | `rgb(var(--ctp-blue) / 0.12)` |
| delete | `--ctp-red` | `rgb(var(--ctp-red) / 0.12)` |

Source: Existing badge pattern from `compliance/page.tsx` FlagBadge component. Consistent with how the payroll and compliance pages signal status.

## Component Inventory

### Page: `/audit-log`

Route: `(dashboard)/audit-log/page.tsx`
Nav entry: Add to `layout.tsx` navItems — label "Audit-Log", icon `ClipboardList` from lucide-react, `adminOnly: true`, `parentViewerVisible: false`.

#### Layout structure

```
<div className="space-y-6">
  <PageHeader />          // h1 + subtitle
  <FilterBar />           // bg-card rounded-xl border border-border p-4
  <ResultsBadge />        // text-xs text-muted-foreground
  <AuditTable />          // bg-card rounded-xl border border-border overflow-hidden
  <PaginationBar />       // text-sm, prev/next buttons
</div>
```

#### PageHeader

- `<h1 className="text-2xl font-bold text-foreground">Audit-Log</h1>`
- `<p className="text-sm text-muted-foreground mt-0.5">Unveränderliche Aufzeichnung aller Schreiboperationen</p>`
- No action button (read-only page — no primary CTA in the header)

#### FilterBar

Three filter controls in `flex flex-wrap gap-3 items-end`:

1. **Entity-Typ** — `<select>` with options: Alle, Dienst, Mitarbeiter, Abrechnung, Abwesenheit, Vertragshistorie
   - Class: `px-3 py-1.5 rounded-lg border border-border bg-background text-foreground text-sm`
2. **Von** — `<input type="date">` — same class as entity-type select
3. **Bis** — `<input type="date">` — same class as entity-type select
4. **Zurücksetzen** button — only visible when any filter is active
   - Class: `text-xs text-muted-foreground hover:text-foreground px-2 py-1.5`

Source: Exact pattern from `compliance/page.tsx` filter section.

#### AuditTable

Columns (left to right):

| Column | Width hint | Content |
|--------|-----------|---------|
| Zeitpunkt | whitespace-nowrap | `dd.MM.yyyy HH:mm` format (de locale) |
| Benutzer | — | User display name or email |
| Aktion | fixed narrow | Badge: `create` / `update` / `delete` with color per action-type map above |
| Entität | — | Entity type label (German) + entity ID as `text-xs text-muted-foreground` below |
| Details | expand icon | Expand button — `ChevronDown` / `ChevronUp` from lucide-react |

Table header row: `border-b border-border` with `text-xs text-muted-foreground` in each `<th className="text-left px-4 py-3">`.

Table body rows:
- `border-b border-border last:border-0 hover:bg-muted/40 transition-colors`
- Alternating row tint on odd rows: `style={idx % 2 === 1 ? { backgroundColor: "rgb(var(--ctp-surface0) / 0.3)" } : {}}`
- Source: Exact pattern from `compliance/page.tsx` table body.

#### Expanded diff row (AUDIT-03 requirement)

When the user clicks the expand button on a row with action `update`, a sub-row slides open below it spanning all columns. It renders a before/after diff for payroll fields.

Structure:

```
<td colSpan={5} className="px-4 pb-3 pt-0">
  <div className="bg-background rounded-lg border border-border p-3 text-xs font-mono space-y-1">
    <DiffLine field="actual_hours" before={...} after={...} />
    <DiffLine field="base_wage"    before={...} after={...} />
    <DiffLine field="total_gross"  before={...} after={...} />
  </div>
</td>
```

DiffLine layout — three parts on one line:
- Field name: `text-muted-foreground w-28 shrink-0`
- Before value: `line-through` + `rgb(var(--ctp-red))` color
- Arrow separator: `→` in `text-muted-foreground mx-2`
- After value: `rgb(var(--ctp-green))` color

Non-payroll `update` rows (employee, absence, shift) show a simplified "Felder geändert: X" label without a full diff table — the structured before/after JSON is available from the API but rendering all fields is out of scope for v1.

#### Loading state

```
<div className="flex items-center justify-center h-32 text-muted-foreground text-sm gap-2">
  <Loader2 size={16} className="animate-spin" /> Wird geladen…
</div>
```

Source: Identical to compliance page loading pattern.

#### Empty state

Icon: `ClipboardList` size={24} color `rgb(var(--ctp-blue))`
Container: `w-12 h-12 rounded-2xl` with `rgb(var(--ctp-blue) / 0.12)` background

Heading: `Keine Einträge gefunden`
Body: `Passe die Filter an oder wähle einen anderen Zeitraum`

Source: Pattern from `compliance/page.tsx` zero-violations empty state.

#### PaginationBar

```
<div className="flex items-center justify-between text-sm text-muted-foreground">
  <span>{offset + 1}–{offset + results.length} von {total}</span>
  <div className="flex gap-2">
    <button disabled={page === 1} ...>
      <ChevronLeft size={16} /> Zurück
    </button>
    <button disabled={isLastPage} ...>
      Weiter <ChevronRight size={16} />
    </button>
  </div>
</div>
```

Page size: 50 rows per page (matches typical audit log ergonomics; no user-configurable page size in v1).
Disabled state: `opacity-50 cursor-not-allowed`

## Interaction Contract

### Filter behavior

- Filters are applied immediately on change (no separate "Suchen" button) — triggers TanStack Query refetch with updated params.
- The "Zurücksetzen" button appears only when at least one filter has a non-empty value.
- Changing any filter resets pagination to page 1.

### Expand/collapse row

- Only one row can be expanded at a time (accordion behavior).
- Clicking the expand button on an already-expanded row collapses it.
- Only rows with action `update` show a diff sub-row; `create` and `delete` rows show no expand button.

### Navigation

- The page is admin-only: `adminOnly: true` in navItems and enforced by `AdminUser` dependency on the backend endpoint.
- No back-navigation or detail page needed — all information is inline.
